## 🚀 Key Learnings So Far
# 🌟 Overall Key Learning – Week 1

Week 1 focused on building a strong foundation for modern software development by combining system-level understanding with practical tooling.

- Gained clarity on why Linux/Unix systems are preferred for development and how WSL enables seamless Linux usage on Windows.
- Understood core operating system concepts including boot processes, virtualization, hypervisors, and cloud infrastructure.
- Developed confidence using the Linux terminal, filesystem navigation, permissions, and system commands.
- Set up a professional development environment using WSL, Visual Studio Code, Python tooling, and package managers.
- Learned Git and GitHub as real-world workflows for version control, collaboration, authentication, and project management.
- Integrated AI tools and APIs directly into the development workflow using terminal-based tooling.
- Adopted a developer mindset focused on reproducibility, efficiency, and scalability.

Overall, Week 1 transformed the system from a regular machine into a production-ready development environment and established the mindset required for advanced software, data science, and AI development.


---

# 🌟 Overall Key Learning – Week 2

Week 2 focused on moving beyond setup and tooling to **building, deploying, and understanding real-world systems** using modern cloud-native practices.

- Gained a clear understanding of **containerization** and how containers differ from virtual machines, including why containers are lightweight, fast, and ephemeral by default.
- Developed hands-on confidence using **Docker and Podman**, learning about images vs containers, volumes for data persistence, networking, and common WSL-related issues.
- Learned to design and build **production-ready backend services with FastAPI**, including REST APIs, data validation, file uploads, and API testing.
- Understood how **frontend and backend systems are separated** in modern architectures, including static hosting constraints, HTTP communication, and CORS configuration.
- Explored **cloud and serverless platforms** such as Vercel and Hugging Face Spaces, understanding their deployment models, limitations, and Git-based workflows.
- Built and deployed **containerized AI and LLM-backed applications**, gaining insight into architectures used in real production environments.
- Strengthened **DevOps and automation fundamentals** through GitHub Codespaces, Dev Containers, and CI/CD pipelines using GitHub Actions.
- Adopted a **system-building, production-first mindset** focused on scalability, reproducibility, automation, and end-to-end delivery.

Overall, Week 2 marked the transition from *writing code* to *shipping systems*, providing practical experience with containers, cloud deployment, automation, and modern backend architecture

# 🌟 **Overall Key Learning – Week 3**

Week 3 emphasized evolving from basic AI usage to **building structured, scalable AI systems** with a strong focus on real-world implementation and reliability.

- Built a clear understanding of how LLMs function internally (tokens, transformers, prediction-based outputs).
- Recognized token usage as a key factor in both system efficiency and cost control.
- Learned how embeddings power semantic understanding and enable intelligent search systems.
- Applied RAG (Retrieval-Augmented Generation) to improve response accuracy and reduce hallucinations.
- Designed semantic search pipelines using chunking, cosine similarity, and vector storage.
- Explored combining keyword and semantic search for more robust and practical retrieval systems.
- Gained exposure to multimodal AI, working with both text and image-based inputs.
- Implemented structured outputs and function calling to integrate AI into automated workflows.
- Strengthened API integration skills and learned how to manage conversational context in applications.
- Understood the importance of testing, evaluation, and proper system setup for reliable deployments.

Overall, Week 3 marked a shift toward thinking like an AI engineer—focusing on system design, reliability, and building end-to-end intelligent applications.
